1
Fix TrustedHosts — Run as Administrator One-time fix on your jump server · Allows WinRM to connect to 10.168.0.32
⚠️ Open PowerShell as Administrator (right-click → Run as Administrator). These commands WILL fail if run in a normal PowerShell window.
Run on YOUR jump server — PowerShell as Administrator 
# Step 1a — Check current TrustedHosts
Get-Item WSMan:\localhost\Client\TrustedHosts

# Step 1b — Add SAP server to trusted list
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "10.168.0.32" -Force

# Step 1c — Confirm it was added
Get-Item WSMan:\localhost\Client\TrustedHosts
# Should show: 10.168.0.32

# Step 1d — Restart WinRM service
Restart-Service WinRM -Force
Write-Host "WinRM restarted — ready to connect" -ForegroundColor Green
💡 If the server already has entries in TrustedHosts, use -Value "existing,10.168.0.32" to append instead of replacing.

Connect — Enter-PSSession Try Method A first. Fall back to Method B or C if it fails.
Method A — NTLM Authentication (most common for cross-domain) 
# Get credentials — a dialog will pop up

# Enter: domain\username  or  .\localadmin
# Example: cre\c5406751  or  .\Administrator

# Connect using NTLM

                -Credential $cred `

# SUCCESS looks like:  [10.168.0.32]: PS C:\Users\...>
Method B — Skip SSL checks (if certificate errors)

    -SkipCACheck `
    -SkipCNCheck `
    -SkipRevocationCheck

Enter-PSSession -ComputerName 10.168.0.32 `
                -Credential $cred `
                -Authentication Negotiate `
                -SessionOption $sessionOpts

                -Credential $cred `
                -Authentication NTLM `
                -Port 5985

3
Fix Winlogon Registry — ROOT CAUSE of RDP failure Run these AFTER you are connected inside the PSSession
🔴 This is the actual break. KB5074109 corrupted the Winlogon Shell and Userinit values. RDP connects but immediately drops because Windows cannot load the user shell. These 3 commands fix it.
Winlogon Registry Fix — Run inside PSSession on 10.168.0.32 
# ── CHECK CURRENT VALUES FIRST ──────────────────────────────
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" `
    | Select-Object Shell, Userinit

# Shell should be:    explorer.exe
# Userinit should be: C:\Windows\system32\userinit.exe,
# If wrong — run the fix below

# ── FIX SHELL ────────────────────────────────────────────────
Set-ItemProperty `
    -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" `
    -Name "Shell" `
    -Value "explorer.exe"

# ── FIX USERINIT ─────────────────────────────────────────────
Set-ItemProperty `
    -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" `
    -Name "Userinit" `
    -Value "C:\Windows\system32\userinit.exe,"

# ── VERIFY FIX APPLIED ───────────────────────────────────────
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" `
    | Select-Object Shell, Userinit

Write-Host "Winlogon registry fixed!" -ForegroundColor Green
❌ BAD Values (broken)
Shell    : cmd.exe
         or blank
Userinit : (blank or wrong path)
✅ CORRECT Values (after fix)
Shell    : explorer.exe
Userinit : C:\Windows\system32\userinit.exe,
           (note trailing comma)
4
Confirm RDP Enabled + Restart Services Run inside PSSession — ensures RDP service is active after registry change
RDP Check + Service Restart — Inside PSSession 
# ── CONFIRM RDP IS ENABLED ───────────────────────────────────
$rdp = Get-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Control\Terminal Server"
Write-Host "fDenyTSConnections = $($rdp.fDenyTSConnections)"
# Must be 0 — if it's 1, RDP is disabled

# ── ENABLE RDP IF DISABLED ───────────────────────────────────
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Terminal Server" `
                 -Name "fDenyTSConnections" -Value 0

# ── ALLOW RDP THROUGH WINDOWS FIREWALL ──────────────────────
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"

# ── RESTART REMOTE DESKTOP SERVICES ─────────────────────────
Stop-Service  -Name TermService -Force
Start-Sleep   -Seconds 4
Start-Service -Name TermService

# ── VERIFY SERVICE IS RUNNING ────────────────────────────────
Get-Service TermService | Select-Object Name, Status

Write-Host "RDP services restarted — test connection now!" -ForegroundColor Green

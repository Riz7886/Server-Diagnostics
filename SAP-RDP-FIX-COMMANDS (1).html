CURRENT ERROR: "Initial Program Was Starting" — RDP connects but login fails NEW
Azure Run Command FIRST — No Admin Needed
Registry Shell + Userinit are CORRECT. TermService is RUNNING. Problem is a broken InitialProgram setting on RDP-Tcp. Fix it via Azure Run Command.
STEP 1 — Paste in Azure Portal → VM → Run Command → RunPowerShellScript:
# Clear broken initial program settings reg delete "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AppSetup /f reg delete "HKCU\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Shell /f reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" /v InitialProgram /t REG_SZ /d "" /f reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" /v fInheritInitialProgram /t REG_DWORD /d 0 /f reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" /v fInheritReconnectSame /t REG_DWORD /d 0 /f # Restart RDP service net stop "Remote Desktop Services" net start "Remote Desktop Services" Write-Host "DONE - Try RDP now"
STEP 2 — After Run Command completes, try RDP from jumphost:
mstsc /v:10.168.0.32
If Azure Run Command still black screen — use Invoke-Command from jumphost normal PowerShell:
Invoke-Command -ComputerName 10.168.0.32 -Credential (Get-Credential -UserName "CRE\c5406751") -ScriptBlock { reg delete "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AppSetup /f reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" /v InitialProgram /t REG_SZ /d "" /f reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" /v fInheritInitialProgram /t REG_DWORD /d 0 /f net stop "Remote Desktop Services" net start "Remote Desktop Services" Write-Host "DONE" }

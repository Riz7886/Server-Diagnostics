STEP 2 💻 LOCAL LAPTOP — PowerShell ⭐ KEY STEP

Start File Server on Local Laptop (Node.js)

💡 Why we do this

RDP copy/paste is blocked. OneDrive is blocked on Jumphost. This turns your laptop into a temporary file server so the Jumphost can pull files directly. Node.js is already installed with Git — no extra software needed.

ℹ️
Your local laptop IP: 192.168.1.79 — this is used in Step 3. If your IP changes run ipconfig on your laptop to get the new IPv4 address under Wi-Fi adapter.

Open PowerShell on LOCAL LAPTOP and run:

# Step 1 — go to your patch folder
cd C:\Users\c5406751\mar-2026

# Step 2 — start the file server using Node.js
node -e "require('http').createServer((q,r)=>{let f=require('fs');f.readFile('.'+q.url,(e,d)=>{r.end(d)})}).listen(8080,()=>console.log('Server running on port 8080!'))"
✅
You will see: Server running on port 8080! — that means it is working. Leave this window open and do NOT close it until all files download on the Jumphost.
🔴
DO NOT CLOSE this PowerShell window until Step 3 is fully complete on the Jumphost. You will see activity in this window as files download.
⚠️
If node is not found: Open Git Bash instead and run the same commands. Git Bash has Node.js included.
STEP 3 🖥️ JUMPHOST — PowerShell as Admin ⭐ KEY STEP

Download All 4 Patch Files to Jumphost

🔴
Do Step 2 FIRST. The Node.js server on your laptop must be running before these commands will work. You must see Server running on port 8080! before continuing.

Open PowerShell as Administrator on the JUMPHOST and paste this entire block:

# Create folder on Jumphost
New-Item -ItemType Directory -Path "C:\Users\c5406751\mar-2026" -Force

# Go into the folder
cd C:\Users\c5406751\mar-2026

# Download file 1 of 4
Invoke-WebRequest "http://192.168.1.79:8080/GoogleChromeStandaloneEnterprise64.msi" -OutFile "GoogleChromeStandaloneEnterprise64.msi"

# Download file 2 of 4
Invoke-WebRequest "http://192.168.1.79:8080/MicrosoftEdgeEnterpriseX64.msi" -OutFile "MicrosoftEdgeEnterpriseX64.msi"

# Download file 3 of 4
Invoke-WebRequest "http://192.168.1.79:8080/mpam-fe.exe" -OutFile "mpam-fe.exe"

# Download file 4 of 4
Invoke-WebRequest "http://192.168.1.79:8080/windows10.0-kb5078752-x64_d19dce98b.msu" -OutFile "windows10.0-kb5078752-x64_d19dce98b.msu"

After all 4 finish — verify they are all there:

dir C:\Users\c5406751\mar-2026
✅
You should see all 4 files listed with correct sizes. Then go back to your local laptop PowerShell and press Ctrl+C to stop the Node.js server.
⚠️
Downloads are slow — that is normal. Files are large (1.31 GB total). Let each one finish before the next starts. Do not close the window.
⚠️
If Invoke-WebRequest fails or times out: Check that your local laptop PowerShell still shows the server running. Check that VPN is connected. Try pinging your laptop from the Jumphost: ping 192.168.1.79
STEP 4 🔒 JUMPHOST → SecureFX → IL5 SFTP

Transfer Patches via SecureFX to IL5 SFTP Server

🔒
SecureFX connects to: sftp-inbound.sms.il5.sapns2.internal using your IL5 password. This is the only approved method to push files into the IL5 environment.

4a — Open SecureFX on the Jumphost — LEFT panel navigate to:

C:\Users\c5406751\mar-2026

You should see your 4 patch files in the LEFT panel.

4b — RIGHT panel (IL5 SFTP server) navigate to:

/il5-sms-scanned-artifacts/s4pce/C5406751
🔴
Must be in YOUR folder! Right panel must show C5406751 — your user ID. Do NOT write to other user folders like i556346 or i746673.

4c — Create mar-2026 folder on RIGHT panel:

  1. Right-click in empty space on the RIGHT panel
  2. Click New Folder → type mar-2026 → Enter
  3. Double-click mar-2026 to open it

4d — Upload the 4 files:

  1. Click anywhere in the LEFT panel → press Ctrl+A to select all 4 files
  2. Press F5 to upload — OR drag them to the RIGHT panel
  3. Click OK/Upload to confirm
  4. Watch the Transfer Queue at the bottom — wait for all 4 to show Done ✅
✅
Verify: RIGHT panel shows all 4 files inside /il5-sms-scanned-artifacts/s4pce/C5406751/mar-2026/ with today's date.
⚠️
If SecureFX session timed out: Re-enter your IL5 password to reconnect. Check VPN is still active.

ALWAYS DO THIS FIRST - SETUP

1
Open PowerShell as Administrator on ns2sw1app1

Right-click PowerShell and select Run as Administrator. Then run:

Set-ExecutionPolicy Bypass -Scope Process -Force cd C:\temp\cert-sw1

SCENARIO A - CHECK WHAT IS IN THE CERT STORE

Run this first every time to see the current state of the certificate store.

.\Fix-RDP-Cert-NS2W1.ps1 -Step VALIDATE
If it shows Total certs in store: 0 — the cert is not imported yet. Go to Scenario B or C below.

SCENARIO B - IMPORT EXISTING PFX FILE (NO PASSWORD)

The PFX file sw1.pfx already exists in C:\temp\cert-sw1\. If it was created without a password use this command to import it into the machine certificate store.

Option 1 - Import with no password (try this first)

$pwd = ConvertTo-SecureString -String "" -Force -AsPlainText Import-PfxCertificate -FilePath "C:\temp\cert-sw1\sw1.pfx" -CertStoreLocation Cert:\LocalMachine\My -Password $pwd

Option 2 - Import with a password (if Option 1 fails)

$pwd = Read-Host "Enter PFX password" -AsSecureString Import-PfxCertificate -FilePath "C:\temp\cert-sw1\sw1.pfx" -CertStoreLocation Cert:\LocalMachine\My -Password $pwd

After importing - validate immediately

.\Fix-RDP-Cert-NS2W1.ps1 -Step VALIDATE
After import, if VALIDATE shows the cert is there but key size is wrong or SAN is missing, go to Scenario D to generate a fresh CSR instead.

SCENARIO C - FIND THE CERT IF IT IS IN THE WRONG STORE

Sometimes the cert is in the Current User store instead of Local Machine store. Run these to find it.

Check all stores for any cert with sw1 in the name

# Check Local Machine store Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.Subject -like "*sw1*" -or $_.FriendlyName -like "*sw1*" } # Check Current User store Get-ChildItem Cert:\CurrentUser\My | Where-Object { $_.Subject -like "*sw1*" -or $_.FriendlyName -like "*sw1*" } # Show ALL certs in Local Machine store Get-ChildItem Cert:\LocalMachine\My | Select-Object FriendlyName, Subject, NotAfter, Thumbprint

If cert is in Current User store - move it to Local Machine

# Get the thumbprint from the output above then run this $cert = Get-ChildItem Cert:\CurrentUser\My | Where-Object { $_.Subject -like "*sw1*" } $store = New-Object System.Security.Cryptography.X509Certificates.X509Store("My","LocalMachine") $store.Open("ReadWrite") $store.Add($cert) $store.Close() Write-Host "Cert moved to LocalMachine\My"

SCENARIO D - GENERATE BRAND NEW 2048-BIT CSR

Use this if the existing cert is invalid, expired, wrong key size, or missing SAN. This creates a fresh private key and CSR on the server.

.\Fix-RDP-Cert-NS2W1.ps1 -Step GENERATECSR

This creates two files:

  • C:\temp\cert-sw1\ns2sw1app1-request.inf — the configuration file
  • C:\temp\cert-sw1\ns2sw1app1-CSR.txt — send this to the CA for signing
After running GENERATECSR: Send the file ns2sw1app1-CSR.txt to the Certificate Authority. Wait for the signed .cer file to come back before running the IMPORT step.

Manually generate CSR if the script fails

certreq -new "C:\temp\cert-sw1\sw1_csr.inf" "C:\temp\cert-sw1\ns2sw1app1-CSR.txt"

SCENARIO E - IMPORT SIGNED CERTIFICATE FROM CA AND BIND EVERYTHING

Run this once the CA returns the signed certificate file. Replace signed.cer with the actual filename.

.\Fix-RDP-Cert-NS2W1.ps1 -Step IMPORT -SignedCertPath "C:\temp\cert-sw1\signed.cer"

This automatically does everything:

  1. Imports the signed cert and pairs with private key using certreq -accept
  2. Sets friendly name to cert-sw1
  3. Removes old IIS HTTPS binding
  4. Creates new IIS HTTPS binding on port 443
  5. Binds cert to RDP via WMI
  6. Falls back to registry if WMI fails
  7. Restarts RDP service (TermService)
  8. Runs full validation and prints PASS or WARN

Manually import signed cert if script fails

certreq -accept "C:\temp\cert-sw1\signed.cer"

SCENARIO F - MANUALLY BIND CERT TO RDP IF SCRIPT DOES NOT DO IT

If the cert is imported but RDP still shows invalid cert, bind it manually.

Get the thumbprint of the cert first

Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.FriendlyName -like "*sw1*" -or $_.Subject -like "*sw1*" } | Select-Object Thumbprint, Subject, FriendlyName

Bind to RDP via WMI using thumbprint from above

# Replace THUMBPRINTHERE with the actual thumbprint from above $tp = "THUMBPRINTHERE" $TSSettings = Get-WmiObject -Namespace root\cimv2\TerminalServices -Class Win32_TSGeneralSetting $TSSettings.SSLCertificateSHA1Hash = $tp $TSSettings.Put() Write-Host "RDP cert bound successfully"

Restart RDP service after binding

Restart-Service TermService -Force Write-Host "RDP service restarted"

Verify RDP binding worked

$rdp = Get-WmiObject -Namespace root\cimv2\TerminalServices -Class Win32_TSGeneralSetting Write-Host "RDP Cert Thumbprint: $($rdp.SSLCertificateSHA1Hash)"

SCENARIO G - MANUALLY BIND CERT TO IIS PORT 443

If IIS HTTPS binding needs to be set manually.

Get thumbprint first then bind to IIS

Import-Module WebAdministration # Get thumbprint $cert = Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.FriendlyName -like "*sw1*" } | Select-Object -First 1 $tp = $cert.Thumbprint Write-Host "Thumbprint: $tp" # Remove old binding if exists Get-WebBinding -Name "Default Web Site" -Protocol "https" | Remove-WebBinding # Create new binding New-WebBinding -Name "Default Web Site" -Protocol "https" -Port 443 -IPAddress "*" $binding = Get-WebBinding -Name "Default Web Site" -Protocol "https" $binding.AddSslCertificate($tp, "My") Write-Host "IIS binding complete"

If WebAdministration fails use netsh instead

# Get thumbprint first $cert = Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.FriendlyName -like "*sw1*" } | Select-Object -First 1 $tp = $cert.Thumbprint $guid = [System.Guid]::NewGuid().ToString("B") # Remove old binding netsh http delete sslcert ipport=0.0.0.0:443 # Add new binding netsh http add sslcert ipport=0.0.0.0:443 certhash=$tp appid=$guid Write-Host "netsh binding complete"

SCENARIO H - EXPORT PFX BACKUP AFTER CERT IS WORKING

Once the cert is installed and working, export a PFX backup file with a password.

# Export PFX with password certutil -exportPFX -p "YourPasswordHere" My cert-sw1 "C:\temp\cert-sw1\cert-sw1-backup.pfx" Write-Host "PFX backup saved to C:\temp\cert-sw1\cert-sw1-backup.pfx"

Alternative - Export using PowerShell

$cert = Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.FriendlyName -like "*sw1*" } | Select-Object -First 1 $pwd = ConvertTo-SecureString -String "YourPasswordHere" -Force -AsPlainText Export-PfxCertificate -Cert $cert -FilePath "C:\temp\cert-sw1\cert-sw1-backup.pfx" -Password $pwd Write-Host "PFX exported successfully"

SCENARIO I - FULL MANUAL VALIDATION - CONFIRM EVERYTHING IS WORKING

Run all of these to confirm the cert is fully installed and bound correctly.

# 1. Check cert is in store Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.FriendlyName -like "*sw1*" -or $_.Subject -like "*sw1*" } | Select-Object FriendlyName, Subject, NotAfter, Thumbprint # 2. Check key size is 2048 $cert = Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.FriendlyName -like "*sw1*" } | Select-Object -First 1 Write-Host "Key Size: $($cert.PublicKey.Key.KeySize)-bit" Write-Host "Has Private Key: $($cert.HasPrivateKey)" Write-Host "Expiry: $($cert.NotAfter)" # 3. Check RDP binding $rdp = Get-WmiObject -Namespace root\cimv2\TerminalServices -Class Win32_TSGeneralSetting Write-Host "RDP Cert: $($rdp.SSLCertificateSHA1Hash)" # 4. Check IIS binding Import-Module WebAdministration Get-WebBinding -Protocol "https" | Select-Object bindingInformation # 5. Check RDP service is running Get-Service TermService | Select-Object Name, Status
All good when you see: Key Size: 2048-bit, Has Private Key: True, RDP Cert matches thumbprint, IIS binding shows 0.0.0.0:443, TermService Status: Running
